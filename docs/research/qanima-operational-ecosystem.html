Introduction

A public map of a constrained operational ecosystem

This article accompanies the interactive map Achilleion / Q_ANIMA Operational Ecosystem. The map shows seven public-facing branches of the current research architecture: Q_ANIMA Runtime, Multi-Model Council, Macro Runtime Alignment, Meta-Orchestrator and Governance, Memory / NCO / Retrieval, Snapshot Store / Self-Binding, and Future Framework.

The map is not a product announcement. It is a transparency document. It renders visible an architecture that is still in development, built layer by layer with explicit constraints at each step. The goal is not to demonstrate capability in the conventional sense. The goal is to demonstrate a particular kind of epistemic discipline: the ability to observe without claiming, to measure without inferring, and to accumulate evidence without rushing to act.

Most descriptions of AI systems in operational contexts focus on what the system can do. This document focuses on what the system deliberately does not do, and why those constraints are architectural choices rather than limitations.

Governance distinction

Why Achilleion is not merely an agentic system

The vocabulary of AI agents has proliferated rapidly. An agent, in common usage, perceives its environment, plans actions, uses tools and executes tasks toward a goal. Multi-agent systems extend this pattern by distributing subtasks across multiple specialized agents. Achilleion / Q_ANIMA does not fit either description. The differences are structural.

The last row is decisive. In many retrieval-augmented systems, a document that mentions an ID may be treated as equivalent to the canonical document identified by that ID. Achilleion / Q_ANIMA keeps these categories separate. This is not a technical detail; it is a foundational epistemic commitment.

Seven branches

The operational branches, stated conservatively

Q_ANIMA Runtime

Status: active, dry-run, observe-only. Q_ANIMA Runtime is the signal observation and shadow decision layer of the ecosystem. It monitors signals, tracks regime states, observes gate conditions and records shadow decisions as proposals, not executions.

Q_ANIMA is not a trading bot. It does not place orders, execute decisions automatically or authorize public operation. No automatic live trading is active at any layer.

Multi-Model Council

Status: active, observe-only, no gate mutation. The Council is a diagnostic ensemble that compares specialized views to detect convergence or divergence across signal interpretations.

The Council does not mutate operational gates, authorize trades or make autonomous decisions. Its outputs remain diagnostic and subject to human governance.

Macro Runtime Alignment

Status: active, observe-only, not causal. Macro Runtime Alignment checks whether macro-level context shows enough variation to support a meaningful relation audit.

Current macro relation audit is observe-only. It measures observed association, not causality, and does not produce market predictions.

Meta-Orchestrator & Governance

Status: V1.1 active, dry-run, propose-only. The Meta-Orchestrator coordinates modules without acting autonomously. It proposes next actions, checks evidence and remains constrained by Governance Invariants G-01 and Evidence Schema G-01.1.

It does not execute, mutate gates, schedule operational changes or bypass human confirmation.

Memory / NCO / Retrieval

Status: active, Local Retrieval Bridge V1.2, canonical identity guard. NCO entries are consolidated memory nodes, not raw logs. The retrieval bridge separates exact textual matches, canonical document identity and mentions.

A mentioned ID is not a canonical document. The system prevents mentions from being promoted into sources.

Snapshot Store / Self-Binding

Status: active but insufficient for causal audit. The Snapshot Store records system states across time and supports self-binding commitments, but it is not mature enough for causal confirmation.

STEP5 remains locked until sufficient snapshots, macro variation, counterfactual traces, closure tests and human sign-off exist.

Future Framework

Status: planned, not active. Future layers describe a governed roadmap, not current capability. Every future layer remains bound to invariants, evidence, snapshot sufficiency and human confirmation.

Future development does not imply immediate autonomy.

Methodological restraint

The system refused to overclaim

The most epistemically significant event in the current state of Achilleion / Q_ANIMA is not something the system did. It is something the system refused to do.

The Macro Council Relation Audit V1 accumulated observational history across distinct runs. It reached the minimum threshold required to attempt an observe-only relation audit, then stopped when macro context variation was insufficient. A relation measured exclusively within one macro regime cannot be generalized. The system had enough rows to inspect, but not enough contextual variation to support a stronger inference.

This behavior is by design. The evidence schema does not permit an observed association to be promoted into a causal claim when the variation needed to test that relation is absent. Statistical correlation in a homogeneous context is not causality.

Current macro relation audit is observe-only. It measures observed association, not causality. Macro context does not authorize trades. STEP5 remains locked. No operational execution is active in the public layer.
Public / private boundary

What this page publishes, and what it does not publish

This document and the interactive map describe the architecture and principles of Achilleion / Q_ANIMA. They do not expose private runtime code, credentials, exchange integration details, private telemetry, operational state or sensitive identifiers.

The public layer contains research artifacts, architectural descriptions, canonical terminology, conceptual maps, working papers and this interactive ecosystem map. The private layer contains operational implementation details and protected runtime state. References to runtime modules use public conceptual names only.

Conclusion

Persistence, memory and governed transformation

Achilleion / Q_ANIMA does not seek immediate autonomy. It seeks persistence, memory, governance and controlled transformation.

The architecture described here is built around the premise that a system's most important capability is not what it can do, but what it can refuse to do and what it requires before it acts. Governance Invariants, evidence schema, identity guard, snapshot sufficiency and the explicit STEP5 lock are the design.

The current state is an early, careful, observe-only ecosystem. It is accumulating evidence, testing methods, validating retrieval, enforcing invariants and building the conditions under which more can be done safely, verifiably and with human oversight at every step.
